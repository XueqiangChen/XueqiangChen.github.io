1. 为什么要用线程池？

池化技术相比大家已经屡见不鲜了，线程池、数据库连接池、Http 连接池等等都是对这个思想的应用。池化技术的思想主要是为了减少每次获取资源的消耗，提高对资源的利用率。

线程池提供了一种限制和管理资源（包括执行一个任务）。 每个线程池还维护一些基本统计信息，例如已完成任务的数量。

这里借用《Java 并发编程的艺术》提到的来说一下使用线程池的好处：

  • 降低资源消耗。通过重复利用已创建的线程降低线程创建和销毁造成的消耗。
  • 提高响应速度。当任务到达时，任务可以不需要的等到线程创建就能立即执行。
  • 提高线程的可管理性。线程是稀缺资源，如果无限制的创建，不仅会消耗系统资源，还会降低系统的稳定性，使用线程池可以进行统一的分配，调优和监控。

2. 自定义一个简单的线程池

一个线程池应该具备以下要素：

  • 任务队列：用于缓存提交的任务。
  • 任务线程管理功能：一个线程池必须能够很好地管理和控制线程数量，可通过如下三个参数来实现，比如创建线程池时初始的线程数量init；线程池自动扩充时最大的线程数量max；在线程池空闲时需要释放线程但是也要维护一定数量的活跃数量或者核心数量core。有了这三个参数，就能够很好地控制线程池中的线程数量，将其维护在一个合理的范围之内，三者之间的关系是init＜=core＜=max。
  • 任务拒绝策略：如果线程数量已达到上限且任务队列已满，则需要有相应的拒绝策略来通知任务提交者。
  • 线程工厂：主要用于个性化定制线程，比如将线程设置为守护线程以及设置线程名称等。
  • QueueSize：任务队列主要存放提交的Runnable，但是为了防止内存溢出，需要有limit数量对其进行控制。
  • Keepedalive时间：该时间主要决定线程各个重要参数自动维护的时间间隔。

2.5 拒绝策略（DenyPolicy）

DenyPolicy主要用于当Queue中的runnable达到了limit上限时，决定采用何种策略通知提交者。该接口中定义了三种默认的实现。

  1. DiscardDenyPolicy：直接将任务丢弃。
  2. AbortDenyPolicy：向任务提交者抛出异常。
  3. RunnerDenyPolicy：使用提交者所在的线程执行任务。
 1public interface DenyPolicy {
 2
 3	void reject(Runnable runnable, ThreadPool threadPool);
 4
 5	/**
 6	 * 该拒绝策略会直接将任务丢弃
 7	 */
 8	class DiscardDenyPolicy implements DenyPolicy {
 9
10		@Override
11		public void reject(Runnable runnable, ThreadPool threadPool) {
12			//do nothing
13			System.out.println("task will be discard");
14		}
15	}
16
17	/**
18	 * 该拒绝策略会向任务提交者抛出异常
19	 */
20	class AbortDenyPolicy implements DenyPolicy {
21
22		@Override
23		public void reject(Runnable runnable, ThreadPool threadPool) {
24			throw new RunnableDenyException("The Runnable " + runnable + " will be abort.");
25		}
26	}
27
28	/**
29	 * 该拒绝策略会使任务在提交者所在的线程中执行任务
30	 */
31	class RunnerDenyPolicy implements DenyPolicy {
32
33		@Override
34		public void reject(Runnable runnable, ThreadPool threadPool) {
35			if (!threadPool.isShutdown()) {
36				runnable.run();
37			}
38		}
39	}
40}

2.6 InternalTask

InternalTask是Runnable的一个实现，是实际任务存储的数据结构。主要用于线程池内部，该类会使用到RunnableQueue，然后不断地从queue中取出某个runnable，并运行runnable的run方法。

 1public class InternalTask implements Runnable{
 2
 3	private final RunnableQueue runnableQueue;
 4
 5	private volatile boolean running = true;
 6
 7	public InternalTask(RunnableQueue runnableQueue) {
 8		this.runnableQueue = runnableQueue;
 9	}
10
11	@Override
12	public void run() {
13		// 如果当前任务为running并且没有被中断，则其将不断地从queue中获取runnable，然后执行run方法
14        // 这是提交到线程池的任务最终运行的地方
15		while (running && !Thread.currentThread().isInterrupted()) {
16			try {
17				Runnable task = runnableQueue.take();
18				task.run();
19			} catch (InterruptedException e) {
20				running = false;
21				break;
22			}
23		}
24	}
25
26	// 停止当前任务，主要会在线程池的shutdown方法中使用
27	public void stop() {
28		this.running = false;
29	}
30
31}

代码还对该类增加了一个开关方法stop，主要用于停止当前线程，一般在线程池销毁和线程数量维护的时候会使用到。

2.7 线程池详细实现

在LinkedRunnableQueue中有几个重要的属性，第一个是limit，也就是Runnable队列的上限；当提交的Runnable数量达到limit上限时，则会调用DenyPolicy的reject方法；runnableList是一个双向循环列表，用于存放Runnable任务

 1public class LinkedRunnableQueue implements RunnableQueue{
 2
 3	// 任务队列的最大容量，在构造时传入
 4	private final int limit;
 5
 6	// 若任务队列已满，则执行拒绝策略
 7	private final DenyPolicy denyPolicy;
 8
 9	// 存放任务的队列
10	private final LinkedList<Runnable> runnableList = new LinkedList<>();
11
12	private final ThreadPool threadPool;
13
14	public LinkedRunnableQueue(int limit, DenyPolicy denyPolicy, ThreadPool threadPool) {
15		this.limit = limit;
16		this.denyPolicy  = denyPolicy;
17		this.threadPool = threadPool;
18	}
19
20	@Override
21	public void offer(Runnable runnable) {
22		synchronized (runnableList) {
23			if (runnableList.size() >= limit) {
24				// 无法容纳新的任务，执行拒绝策略
25				denyPolicy.reject(runnable, threadPool);
26			} else {
27				// 将任务加入队尾，并且唤醒阻塞中的线程
28				runnableList.addLast(runnable);
29				runnableList.notifyAll();
30			}
31		}
32	}
33
34	/**
35	 * take方法也是同步方法，线程不断从队列中获取Runnable任务，当队列为空的时候工作线程会陷入阻塞，
36	 * 有可能在阻塞的过程中被中断，为了传递中断信号需要在catch语句块中将异常抛出以通知上游（InternalTask）
37	 * @return 任务
38	 * @throws InterruptedException 中断异常，通知上游(InternalTask)
39	 */
40	@Override
41	public Runnable take() throws InterruptedException {
42		synchronized (runnableList) {
43			while (runnableList.isEmpty()) {
44				try {
45					// 如果任务队列中没有可执行任务，则当前线程挂起，进入runnableList关联的monitor waitset中等待唤醒
46					runnableList.wait();
47				} catch (InterruptedException e) {
48					// 被中断时需要将异常抛出
49					throw e;
50				}
51			}
52			// 从任务队列头排除一个任务
53			return runnableList.removeFirst();
54		}
55	}
56
57	@Override
58	public int size() {
59		return runnableList.size();
60	}
61}
62

根据前面的讲解，线程池需要有数量控制属性、创建线程工厂、任务队列策略等功能，线程池初始化代码如下：

  1public class BasicThreadPool extends Thread implements ThreadPool{
  2
  3	// 初始化线程数量
  4	private final int initSize;
  5
  6	// 线程池最大数量
  7	private final int maxSize;
  8
  9	// 线程池核心线程数量
 10	private final int coreSize;
 11
 12	// 当前活跃的线程数量
 13	private int activeCount;
 14
 15	// 创建线程所需的工厂
 16	private final ThreadFactory threadFactory;
 17
 18	// 任务队列
 19	private final RunnableQueue runnableQueue;
 20
 21	// 线程池是否已经被shutdown
 22	private volatile boolean isShutdown = false;
 23
 24	// 工作线程队列
 25	private final Queue<ThreadTask> threadQueue = new ArrayDeque<>();
 26
 27	private final static DenyPolicy DEFAULT_DENY_POLICY = new DenyPolicy.DiscardDenyPolicy();
 28
 29	private final static ThreadFactory DEFAULT_THREAD_FACTORY = new DefaultThreadFactory();
 30
 31	private final long keepAliveTime;
 32
 33	private final TimeUnit timeUnit;
 34
 35	// 构造时需要传递的参数：初始的线程数量，最大的线程数量，核心线程数量，任务队列的最大数量
 36	public BasicThreadPool(int initSize, int maxSize, int coreSize, int queueSize) {
 37		this(initSize, maxSize, coreSize, DEFAULT_THREAD_FACTORY, queueSize, DEFAULT_DENY_POLICY, 10 ,TimeUnit.SECONDS);
 38	}
 39
 40	// 构造线程池时需要传入的参数，该构造函数需要的参数比较多
 41	public BasicThreadPool(int initSize, int maxSize, int coreSize, ThreadFactory threadFactory, int queueSize, DenyPolicy denyPolicy, long keepAliveTime, TimeUnit timeUnit) {
 42		this.initSize = initSize;
 43		this.maxSize = maxSize;
 44		this.coreSize = coreSize;
 45		this.threadFactory = threadFactory;
 46		this.runnableQueue = new LinkedRunnableQueue(queueSize, denyPolicy, this);
 47		this.keepAliveTime = keepAliveTime;
 48		this.timeUnit = timeUnit;
 49		this.init();
 50	}
 51
 52	// 初始化时，先创建 initSize 个线程
 53	private void init() {
 54		start();
 55		for (int i = 0; i < initSize; i++) {
 56			newThread();
 57		}
 58	}
 59
 60	private void newThread() {
 61		//创建任务线程，并且启动
 62		InternalTask internalTask = new InternalTask(runnableQueue);
 63		Thread thread = this.threadFactory.createThread(internalTask);
 64		ThreadTask threadTask = new ThreadTask(thread, internalTask);
 65		threadQueue.offer(threadTask);
 66		this.activeCount++;
 67		thread.start();
 68	}
 69
 70	private void removeThread() {
 71		// 从线程池中移除某个线程
 72		ThreadTask threadTask = threadQueue.remove();
 73		threadTask.internalTask.stop();
 74		this.activeCount--;
 75	}
 76
 77	@Override
 78	public void execute(Runnable runnable) {
 79		if (this.isShutdown) {
 80			throw new IllegalStateException("The thread pool is destroy");
 81		}
 82		// 提交任务只是简单地往任务队列中插入Runnable
 83		this.runnableQueue.offer(runnable);
 84	}
 85
 86	@Override
 87	public void run() {
 88		// run 方法继承自Thread，主要用于维护线程数量，比如扩容、回收工作
 89		while (!isShutdown && !isInterrupted()) {
 90			try {
 91				timeUnit.sleep(keepAliveTime);
 92			} catch (InterruptedException e) {
 93				isShutdown = true;
 94				break;
 95			}
 96
 97			synchronized (this) {
 98				if (isShutdown) {
 99					break;
100				}
101				//当前队列中有尚未处理，并且activeCount<coreSize则继续扩容
102				if (runnableQueue.size() > 0 && activeCount < coreSize) {
103					for (int i = initSize; i < coreSize; i++) {
104						newThread();
105					}
106					// continue 的目的在于不想让线程的扩容直接达到maxsize
107					continue;
108				}
109				// 当前队列中有任务尚未处理，并且activeCount<maxSize则继续扩容
110				if (runnableQueue.size() > 0 && activeCount < maxSize) {
111					for (int i = coreSize; i < maxSize; ++i) {
112						newThread();
113					}
114				}
115
116				// 如果任务队列中没有任务，则需要回收，回收至coreSize即可
117				if (runnableQueue.size() == 0 && activeCount > coreSize) {
118					for (int i = coreSize; i < activeCount; i++) {
119						removeThread();
120					}
121				}
122			}
123		}
124	}
125
126	//ThreadTask 只是InternalTask和Thread的一个组合
127	private static class ThreadTask {
128		Thread thread;
129		InternalTask internalTask;
130
131		public ThreadTask(Thread thread, InternalTask internalTask) {
132			this.thread = thread;
133			this.internalTask = internalTask;
134		}
135	}
136
137	/**
138	 * 销毁线程池主要为了是停止BasicThreadPool线程，停止线程池中的活动线程并且将isShutdown开关变量更改为true。
139	 */
140	@Override
141	public void shutdown() {
142		synchronized (this) {
143			if (isShutdown) {
144				return;
145			}
146
147			isShutdown = true;
148			threadQueue.forEach(threadTask -> {
149				threadTask.internalTask.stop();
150				threadTask.thread.interrupt();
151			});
152
153			this.interrupt();
154		}
155	}
156
157	@Override
158	public int getInitSize() {
159		if (isShutdown) {
160			throw new IllegalStateException("The thread pool is destroy");
161		}
162		return this.initSize;
163	}
164
165	@Override
166	public int getMaxSize() {
167		if (isShutdown) {
168			throw new IllegalStateException("The thread pool is destroy");
169		}
170		return this.maxSize;
171	}
172
173	@Override
174	public int getCoreSize() {
175		if (isShutdown) {
176			throw new IllegalStateException("The thread pool is destroy");
177		}
178		return this.coreSize;
179	}
180
181	@Override
182	public int getQueueSize() {
183		if (isShutdown) {
184			throw new IllegalStateException("The thread pool is destroy");
185		}
186		return runnableQueue.size();
187	}
188
189	@Override
190	public int getActiveCount() {
191		synchronized (this) {
192			return this.activeCount;
193		}
194	}
195
196	@Override
197	public boolean isShutdown() {
198		return this.isShutdown;
199	}

自动维护线程的代码块是同步代码块，主要是为了阻止在线程维护过程中线程池销毁引起的数据不一致问题。

任务队列中若存在积压任务，并且当前活动线程少于核心线程数，则新建 coreSize-initSize数量的线程，并且将其加入到活动线程队列中，为了防止马上进行maxSize-coreSize数量的扩充，建议使用continue终止本次循环。

任务队列中有积压任务，并且当前活动线程少于最大线程数，则新建maxSize-coreSize数量的线程，并且将其加入到活动队列中。

当前线程池不够繁忙时，则需要回收部分线程，回收到coreSize数量即可，回收时调用removeThread()方法，在该方法中需要考虑的一点是，如果被回收的线程恰巧从Runnable任务取出了某个任务，则会继续保持该线程的运行，直到完成了任务的运行为止，详见InternalTask的run方法。
