队列及其实现

Aug 6, 2020 · 3 min read · 队列  ·
Share on:

1. 什么是队列

具有一定操作约束的线性表。插入和删除操作，只能在一端插入，另一端删除。

数据插入称之为入队(addQ)，数据删除称之为出队(deleteQ)，队列最重要的特征就是先进先出(FIFO)。生活中有很多跟队列相关的例子，例如超市排队。

2. 队列的抽象数据类型描述

与队列相关的操作主要包括以下几种：

  • 创建队列：生成长度为 size 的空队列。
  • 判断队列是否满了。
  • 判断队列是否为空。
  • 将数据元素插入到队列中。
  • 将数据元素从队列中删除。

3. 队列的顺序存储实现

队列的顺序存储结构通常由一个一维数组和一个记录队列头元素位置的变量front以及一个记录队列尾元素位置的变量rear组成。

如下图所示，用顺序存储实现队列，由于数组的元素从 0 开始，所以 front 和 rear 同时指向 -1 这个位置，添加 Job1，rear 往后移动一个位置，删除 Job1 ，front 往后移动一个位置。当队列满了的时候，就无法添加元素了，但是很明显就能发现，此时之前删除的位置还是空的，队列中还有位置，只是无法添加而已，这样的结构会造成空间浪费，我们需要用循环结构来解决。

循环队列的机构如下图所示。循环结构中，front 和 rear 开始时同时指向 0 这个位置，之后，每一次入队，rear 向着顺时针方向移动一个位置，每一次出队列，front 向顺时针方向移动一个位置。那么这里就有个问题：队列空和满的判别条件是什么？队列空和满的时候，front=rear，那么就造成无法判断队列空还是满了。那么要如何解决呢？这里提供两个解决方法：

  1. 使用额外标记： Size或者tag 。size 用来记录当前元素的个数，当你加入一个元素的时候，size 加 1，删除一个元素的时候，size减1，所以只要根据size是0还是n就知道是空还是满的。tag （0，1）标记，添加一个元素，tag=1，删除一个元素 tag=0，当我们想判断队列是满还是空时，只要判断 tag 的值就知道最后一次操作是添加还是删除。
  2. 仅使用n-1个数组空间。

我们采用第二种方案，使用求余函数来查看列队是否已满。

1(rear + 1) % size == front

看看具体的实现代码：

 1package leetcode.editor.datastructure.queue;
 2
 3import java.io.Serializable;
 4
 5public class SeqQueue<T> implements Queue<T>, Serializable {
 6
 7    private final static int DEAFULT_SIZE = 10;
 8
 9    private T elementData[];
10
11    private int front, rear;
12
13    private int size;
14
15    public SeqQueue() {
16        elementData = (T[]) new Object[DEAFULT_SIZE];
17        front = 0;
18        rear = 0;
19    }
20
21    public SeqQueue(int size) {
22        this.size = size;
23        elementData = (T[]) new Object[size];
24        front = 0;
25        rear = 0;
26    }
27
28    public void add(T data) {
29        if ((rear + 1) % this.elementData.length == front) {
30            System.out.println("队列已满");
31            return;
32        }
33        rear = (rear + 1) % this.elementData.length;
34        elementData[rear] = data;
35    }
36
37    public boolean isEmpty() {
38        return this.elementData.length == 0;
39    }
40
41    public T delete() {
42        if (front == rear) {
43            System.out.println("队列为空");
44            return null;
45        } else {
46            front = (front + 1) % this.elementData.length;
47            return elementData[front];
48        }
49    }
50}

4. 队列的链式存储实现

队列的链式存储结构也可以用一个单链表实现。插入和删除操作分别在链表的两头进行；队列指针front和rear应该分别指向链表的表头和表尾。

整个队列的结构如下图所示：

与顺序结构不同的是，链式存储实现的队列，出队需要在表头进行，因为是单向链表，如果在表尾进行删除操作，我们无法知道前一个元素是多少。因此入队和出队操作为：

 1public class LinkedQueue<T> implements Queue<T> {
 2
 3    private Node<T> front; //指向队头节点
 4
 5    private Node<T> rear; //指向队尾节点
 6
 7    public LinkedQueue() {
 8        this.front = null;
 9        this.rear = null;
10    }
11
12    @Override
13    public void add(T data) {
14        Node<T> node = new Node<>(data, null);
15        if (this.front == null) {//空队列插入
16            this.front = node;
17        } else {//非空队列,尾部插入
18            this.rear.next = node;
19        }
20        this.rear = node;
21    }
22
23    @Override
24    public boolean isEmpty() {
25        return front == null && rear == null;
26    }
27
28    @Override
29    public T delete() {
30        Node<T> frontCell;
31        T frontElem;
32
33        if (this.front == null) {
34            System.out.println("队列为空");
35            return null;
36        }
37        frontCell = front;
38        if (front == rear) //若队列只有一个元素
39            front = rear = null; //删除后队列置为空
40        else
41            front = front.next();//front移动到下一个元素
42        frontElem = frontCell.data;
43        return frontElem;
44    }
45}
46
47public class Node<T> {
48
49    public T data;
50    public Node<T> next;
51
52    public Node(T data) {
53        this.data = data;
54        next = null;
55    }
56
57    public Node(T data, Node<T> next) {
58        this.data = data;
59        this.next = next;
60    }
61
62    public Node<T> next() {
63        return this.next;
64    }
65
66}
